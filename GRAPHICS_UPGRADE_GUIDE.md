# Ayodhya Protocol — Graphics Upgrade Guide

## Current State

The game currently uses flat-colour primitive meshes (boxes, cylinders) with billboard enemy sprites. PBR textures were generated by Gemini but disabled because they caused a white-screen when they failed to load (the PBRMaterial `albedoColor` defaults to white). The textures already exist in `client/public/textures/` but aren't loaded.

## Strategy: 3 Phases

### Phase 1 — Fix Existing Textures (Code Change, No New Assets)
Re-enable TextureLoader with per-texture error handling so bad textures don't white-screen the entire game. This instantly gives you PBR ground, pillars, trees, and characters.

### Phase 2 — Generate Better Assets with Gemini
Replace the existing Gemini textures with higher-quality versions using the prompts below.

### Phase 3 — Add Cinematic Polish with Perplexity Video
Loading screen video, instruction screen background, game over sequences.

---

## Phase 1 — Safe Texture Re-enable

Already implemented in the codebase. The key change: each texture loads in a try/catch with a fallback, and `albedoColor` is set to the flat-colour fallback instead of white, so missing textures still look correct.

- Request **1024x1024** for game textures (resize later if needed)

- For sprites, include "transparent background, centered, single character"

## Critical Tips for Gemini

1. **Transparent backgrounds**: Gemini sometimes generates with white BG even when asked for transparent. If that happens, add "alpha channel, checkerboard visible behind character" to the prompt, or use the game's existing white-background-removal code in World.ts.

2. **Seamless tiling**: If results aren't perfectly seamless, re-prompt with "The four edges must match perfectly when tiled. Show the texture repeated 2x2 to verify seamlessness."

3. **Consistency**: Generate all enemy/player sprites in the same session so the art style stays consistent.

4. **Normal maps**: For even better visual quality, ask Gemini to generate normal maps: "Generate a normal map (purple-blue tones, tangent space) for [description of the surface]. Flat areas should be RGB(128, 128, 255). Bumps shown as color variations."

5. **Size matters**: Generate at the highest resolution possible, then downscale. 1024x1024 minimum for ground textures, 512x512 for sprites.
